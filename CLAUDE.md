# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Bank AI — a single-screen React chat welcome page. Boots to one route (`/`) showing a spinning orb, brand mark, a 2×2 grid of topic cards loaded from a mock API, a fancy chat input, and a footer. There is no chat backend yet; submitting the input and selecting a topic only update local state / log to the console.

## Commands

```bash
npm run dev      # Vite dev server on http://localhost:9999 (port pinned in vite.config.ts)
npm run build    # tsc -b (typecheck) then vite build → dist/
npm run lint     # oxlint (NOT eslint)
npm run preview  # serve the production build
```

No test runner is configured. `npm run build` is the typecheck gate — run it to validate changes.

## Architecture

- **Stack**: Vite 8 + React 19 + TypeScript, Tailwind CSS **v4**, React Router 7, TanStack Query 5, MSW 2, lucide-react icons.

- **Tailwind v4 is CSS-first** — there is no `tailwind.config.js`. The plugin is wired in `vite.config.ts` (`@tailwindcss/vite`). All theme tokens, keyframes, and custom utilities live in `src/index.css` inside `@theme`, `@property`, `@keyframes`, and `@utility` blocks. Brand colors are exposed as `--color-accent-1/2/3` and used via classes like `from-accent-1` / `text-accent-3`.

- **App bootstrap** (`src/main.tsx`): MSW is started **before** React renders, and only in `import.meta.env.DEV`, so production builds never load the mock worker. React mounts inside `QueryClientProvider` → `BrowserRouter`. `App.tsx` is just the route table.

- **Mock API (MSW)**: The app makes a real `fetch('/api/topics')` that MSW intercepts at the Service Worker layer (`public/mockServiceWorker.js`, generated by `npx msw init`). Handlers and seed data live in `src/mocks/handlers.ts` (includes a deliberate `delay(600)` so loading skeletons are visible); the worker is set up in `src/mocks/browser.ts`. To change/add mock endpoints, edit `handlers.ts`.

- **Data flow for topics**: `src/api/topics.ts` (`fetchTopics`) → `src/hooks/useTopics.ts` (TanStack Query wrapper) → `src/components/TopicGrid.tsx`, which renders skeletons while `isPending`, an inline retry on `isError`, and `TopicCard`s on success. The `Topic.icon` field is a string key mapped to a lucide icon component inside `TopicCard.tsx` — adding a new icon means extending both the `Topic['icon']` union in `src/types.ts` and the `ICONS` map in `TopicCard.tsx`.

- **Layout** (`src/pages/WelcomePage.tsx`): a phone-width column (`max-w-[26rem]`, centered) sized to `h-dvh`. The orb/brand/topics sit in a `flex-1 overflow-y-auto` scroll area; the chat input + footer are a `shrink-0` bottom bar pinned to the bottom of the viewport. The topic grid is a fixed `grid-cols-2` (not viewport-responsive) because it lives inside the narrow column.

- **`Orb`** is reused at two sizes via a `size` prop (large on the page, compact inside the chat input). Its spin and the chat input's rotating gradient border are both driven by the registered `@property --angle` custom property animated with the `spin-angle` keyframe; `ChatInput` speeds up the rotation and activates the orb on focus.

## Conventions

- Components are named exports in `src/components/`; pages in `src/pages/`. Path-less relative imports throughout (no path aliases configured).
- Keep styling in Tailwind utility classes; reach for `src/index.css` only for theme tokens, keyframes, and `@utility` definitions that can't be expressed inline.
