# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Bank AI — a polished, phone-width React chat app for a fictional bank, all running off a mock API (no real backend or LLM). Two routes:

- `/` (`WelcomePage`) — a spinning orb, brand mark, "What can I help?", and a grid of topic cards loaded from a mock API. Submitting the chat input starts a fresh conversation and navigates to `/chat`.
- `/chat` (`ChatPage`) — the conversation view. The thread is chosen from the URL: `?topic=<id>` resumes/starts a topic thread (with a bespoke welcome screen when empty), otherwise it's the `general` thread. A `?message=` deeplink seeds the general thread once.

Assistant replies are **canned markdown strings** returned by MSW; the frontend fakes streaming by typewriter-revealing them. Replies can embed rich UI — GFM tables/task-lists, inline colored highlights, and code-split "custom blocks" (charts, action cards, follow-up suggestion pills) — driven entirely by the reply text.

## Commands

```bash
npm run dev      # Vite dev server on http://localhost:9999 (port pinned in vite.config.ts)
npm run build    # tsc -b (typecheck) then vite build → dist/
npm run lint     # oxlint (NOT eslint)
npm run preview  # serve the production build
```

No test runner is configured. `npm run build` is the typecheck gate — run it to validate changes.

## Architecture

- **Stack**: Vite 8 + React 19 + TypeScript, Tailwind CSS **v4**, React Router 7, TanStack Query 5 (server data), Zustand 5 (chat state), MSW 2 (mock API), Zod 4 (block validation), Recharts 3 (charts), react-markdown + remark-gfm + remark-directive (rich replies), sonner (toasts), lucide-react icons.

- **Tailwind v4 is CSS-first** — there is no `tailwind.config.js`. The plugin is wired in `vite.config.ts` (`@tailwindcss/vite`). All theme tokens, keyframes, and custom utilities live in `src/index.css` inside `@theme`, `@property`, `@keyframes`, and `@utility` blocks. Brand colors are `--color-accent-1/2/3`; inline-highlight tones are `--color-tone-positive/negative/warning/info`; surfaces are `--color-ink` / `--color-ink-deep`. Use them via classes like `from-accent-1` / `text-accent-3` / `bg-ink-deep`.

- **App bootstrap** (`src/main.tsx`): MSW is started **before** React renders, and only in `import.meta.env.DEV`, so production builds never load the mock worker. React mounts inside `QueryClientProvider` → `BrowserRouter`. `App.tsx` is the route table plus the always-on `AnimatedBackground` (drifting ambient orbs) and the global sonner `Toaster` (styled glassmorphism, offset to clear the bottom bar).

- **Mock API (MSW)**: The app makes real `fetch`es that MSW intercepts at the Service Worker layer (`public/mockServiceWorker.js`, generated by `npx msw init`). Handlers and all seed/reply data live in `src/mocks/handlers.ts`; the worker is set up in `src/mocks/browser.ts`. Endpoints:
  - `GET /api/topics` → topic cards (`delay(600)` so loading skeletons show)
  - `GET /api/insights` → insight cards for the Insights welcome screen
  - `POST /api/chat` `{ threadId, message }` → `{ reply }`, a full canned markdown string keyed by thread (`delay(700)`). On the `insights` thread the message text is matched against per-insight replies first.

- **Chat state (`src/store/chatStore.ts`, Zustand + persist)**: One `Message[]` per thread, keyed by `ThreadId` (`transfer | cards | savings | security | insights | general`), plus a single `pending` thread id that locks that thread's input while a reply is in flight or streaming. `sendMessage` appends the user message and kicks off `requestReply`; `startGeneralFresh` resets the general thread from one message (used by the home input and `?message=` deeplink); `retryLast` re-requests after an error; `finishStreaming` is called by the typewriter when a reply finishes revealing (clears `pending`, drops `status`). Persisted to **sessionStorage** as `bank-ai-chat` — only `threads` are persisted, and any message left mid-stream is normalized to done so it renders fully on reload.

- **Fake streaming**: `sendChat` returns the whole reply at once. `MessageBubble` marks the new assistant message `status: 'streaming'` and feeds its prose through `useTypewriter` (`src/hooks/useTypewriter.ts`), which reveals ~2 chars/16ms. Atomic spans — `` ```bank:<name>``` `` fences and `:hl[…]{…}` directives — are snapped past in one frame so their raw syntax never flashes. On completion it calls `finishStreaming`.

- **Layout (`src/components/AppShell.tsx`)**: the phone-width column (`mx-auto`, centered, `h-dvh`) shared by both pages — optional header, a scrollable content area, and the `ChatInput` + `Footer` at the bottom. Two modes:
  - default (home): header/content/bottom-bar in normal flex flow.
  - `overlay` (chat): content fills the height and scrolls behind blurred, fading top/bottom bars; the bars' heights are measured with a `ResizeObserver` and fed into the scroll area's padding + `scroll-padding` so the first/last message clears the chrome and scroll-to-bottom lands above the input.

- **Topics → welcome screens**: `src/api/topics.ts` → `src/hooks/useTopics.ts` (TanStack Query) → `src/components/TopicGrid.tsx` (skeletons while `isPending`, inline retry on `isError`, `TopicCard`s on success). Selecting a card navigates to `/chat?topic=<id>`. When a topic thread is empty, `ChatPage` shows a bespoke **welcome screen** resolved in `src/components/welcome/index.tsx` (`TransferWelcome`, `CardServicesWelcome`, `SavingsWelcome`, `SecurityWelcome`, `InsightsWelcome`, falling back to `GenericWelcome`); each takes `{ topic, onSend }` and offers quick-action prompts. `InsightsWelcome` fetches `/api/insights` (`useInsights`) and renders insight cards whose tap sends the insight title as the first message.

- **Rich assistant replies (`src/components/Markdown.tsx`)**: renders reply markdown with `remark-gfm` (tables, task lists, strikethrough, autolinks). **Raw HTML is intentionally not rendered** (no `rehype-raw`), so reply content can't inject markup. Two custom authoring affordances ride on top:
  - **Inline highlights** — `:hl[text]{tone=positive|negative|warning|info}` (a remark text directive mapped to `<Highlight>`; unknown tone → plain text, missing → `info`).
  - **Custom blocks** — `` ```bank:<name>``` `` fenced code whose body is JSON. `CustomBlock` (`src/components/blocks/CustomBlock.tsx`) looks `<name>` up in the **block registry**, `JSON.parse`s + Zod-validates the body, and renders a lazily-imported component; any failure (unknown name, bad JSON, schema mismatch, render error) degrades to an inline `BlockFallback` notice.

- **Block registry (`src/components/blocks/`)**: `registry.ts` maps a block name → `{ schema, lazy Component }` via `defineBlock`, which ties the component's `data` prop to the schema's inferred type at compile time and code-splits the component (keeping Recharts out of the initial bundle). Schemas live in `schemas.ts` (separate from components, so importing the registry never pulls in Recharts). Current blocks: `allocationDonut`, `driftBars` (Recharts), `actionCard`, `suggestions`. **Adding a block = one `defineBlock(...)` line + a schema + a component** under `blocks/`. The `suggestions` block is special-cased: `MessageBubble` uses `splitSuggestions` to hoist it out of the prose and render it (follow-up pills) below the message actions, never inline; pills dispatch back into the current thread via `ChatThreadContext`.

- **`Orb`** is reused at two sizes via a `size` prop (large on the home page, compact inside the chat input). Its spin and the chat input's rotating gradient border are both driven by the registered `@property --angle` custom property animated with the `spin-angle` keyframe; `ChatInput` speeds up the rotation and activates the orb on focus.

- **`Message.icon`-style string-keyed maps**: several types carry a string `icon`/`tone` key resolved to a component/class at render time — `Topic['icon']` → `ICONS` in `TopicCard.tsx`, `Insight['icon']`/`tone` → `InsightCard.tsx`. Adding an option means extending both the union in `src/types.ts` and the corresponding map.

## Conventions

- Components are named exports in `src/components/` (blocks under `components/blocks/`, welcome screens under `components/welcome/`); pages in `src/pages/`. Block components are **default** exports (required by the lazy-import loader). Path-less relative imports throughout (no path aliases configured).
- Keep styling in Tailwind utility classes; reach for `src/index.css` only for theme tokens, keyframes, and `@utility` definitions that can't be expressed inline.
- The look is dark glassmorphism: translucent `white/5–10` surfaces, `border-white/10`, `backdrop-blur`, accent gradients. Match it when adding UI.
